# Milestone M05: ThothSpinner Integration (v0.5.0)

**Status**: [x] COMPLETED

**Goal/Requirement**: Combine all components into unified ThothSpinner orchestrator
- Create parent orchestrator component that manages all child components
- Implement comprehensive state management system (in_progress, success, error)
- Add configuration hierarchy with inheritance
- Support element ordering and layout control
- Implement fade-away animations for state transitions
- Provide unified API for controlling all components
- Support both programmatic and configuration-based initialization

**Out of Scope**:
- Textual widget implementations (M06)
- Documentation and publishing (M07)
- Additional component types beyond the five core components
- Complex layout engines or grid systems
- Persistent configuration storage
- Event systems or callbacks (use direct method calls)

## Implementation Notes (Based on Rich Pattern Analysis)

### Key Design Decisions:
1. **Simplified Registry**: Use dict by short type names + immutable render_order tuple
2. **Hybrid Control**: Both convenience methods and generic update_component()
3. **Pure Renderable**: Implement __rich__() and __rich_measure__() protocols
4. **Keyword-only Args**: All update methods use * to enforce keyword arguments
5. **Direct Calls**: No event system, use direct component method calls
6. **Type-based Access**: get_component("spinner") with KeyError on invalid type
7. **Thread Safety**: Use RLock for concurrent access (matches Rich patterns)
8. **Eager Creation**: All 5 components created on init (like Rich's Status)
9. **Type Safety**: Use ComponentState enum, TypedDict hints, validation

## Deliverable

```python
from thothspinner import ThothSpinner
from rich.console import Console
from rich.live import Live
import time

console = Console()

# Simple initialization with kwargs (Rich pattern)
spinner = ThothSpinner(
    spinner_style="npm_dots",
    message_shimmer=True,
    progress_format="percentage",
    timer_format="auto",
    hint_text="(esc to cancel)",
    success_duration=2.0,  # Auto-clear after 2 seconds
    error_duration=3.0     # Auto-clear after 3 seconds
)

# Or with configuration dict
config = {
    "defaults": {
        "color": "#D97706",
        "visible": True
    },
    "elements": {
        "spinner": {"style": "npm_dots", "success_icon": "✓", "error_icon": "✗"},
        "message": {"shimmer": {"enabled": True, "width": 3}},
        "progress": {"format": {"style": "percentage"}, "total": 100},
        "timer": {"format": {"style": "auto"}},
        "hint": {"text": "(esc to cancel)", "color": "#888888"}
    },
    "render_order": ["spinner", "message", "progress", "timer", "hint"],
    "fade_away": {
        "enabled": True,
        "direction": "left-to-right",
        "interval": 0.05
    }
}

spinner = ThothSpinner(**config)

# Use with Rich Live (ThothSpinner is a pure renderable)
with Live(spinner, console=console, refresh_per_second=20):
    # Start in in_progress state
    spinner.start()

    # Simulate work with progress updates (keyword-only args)
    for i in range(101):
        spinner.update_progress(current=i, total=100)
        time.sleep(0.05)

        # Direct shimmer control (no events)
        if i == 50:
            spinner.set_shimmer_direction(direction="right-to-left")

    # Transition to success state with auto-clear duration
    spinner.success("Task completed successfully!", duration=2.0)
    time.sleep(2.5)  # Will auto-clear after 2 seconds

    # Reset and do another task
    spinner.reset()
    spinner.set_message(text="Processing data")
    time.sleep(2)

    # Generic update method for flexibility
    spinner.update_component("message", text="Almost done...")

    # Demonstrate error state with fade-away
    spinner.error("Connection failed", duration=3.0)
    time.sleep(3.5)  # Auto-clears after 3 seconds

    # Can also use stop() for Rich API compatibility
    spinner.stop()  # Alias for clear()

# Alternative: Direct component access
spinner = ThothSpinner()
try:
    progress = spinner.get_component("progress")
    progress.set(50)  # Direct component manipulation
except KeyError:
    print("Component not found")  # Raised for invalid component types
```
