def _calculate_next_word_change(self, current_time: float) -> bool:
    """Determine if word should change based on elapsed time."""
    if self._last_word_change is None:
        self._last_word_change = current_time
        self._next_interval = random.uniform(self.min_interval, self.max_interval)
        return True
    
    elapsed = current_time - self._last_word_change
    if elapsed >= self._next_interval:
        self._last_word_change = current_time
        self._next_interval = random.uniform(self.min_interval, self.max_interval)
        return True
    return False

### 3. Shimmer Effect Implementation

Based on Rich's pulse animation pattern (`/Users/stevemorin/c/rich/rich/progress_bar.py:126-154`), implement character-by-character styling:

```python
def _apply_shimmer(self, text: str, current_time: float) -> Text:
    """Apply shimmer effect to text using Rich Text spans."""
    # Calculate shimmer position based on time
    elapsed = current_time - self._shimmer_start_time
    total_positions = len(text) + self.shimmer_width * 2
    
    if self.reverse_shimmer:
        position = total_positions - int(elapsed * self.shimmer_speed * 10) % total_positions
    else:
        position = int(elapsed * self.shimmer_speed * 10) % total_positions
    
    # Build styled text with shimmer
    result = Text()
    shimmer_start = position - self.shimmer_width
    
    for i, char in enumerate(text):
        if shimmer_start <= i < shimmer_start + self.shimmer_width:
            # Apply light color for shimmer effect
            result.append(char, style=Style(color=self.shimmer_light_color))
        else:
            # Apply base color
            result.append(char, style=Style(color=self.base_color))
    
    return result
```

**Key Implementation Details**:
- Use `Text.append()` with different styles for each character
- Calculate position based on elapsed time for smooth animation
- Support bidirectional movement with `reverse_shimmer` flag

### 6. Performance Optimizations

Based on Rich's caching patterns (`/Users/stevemorin/c/rich/rich/progress.py:510-542`):

```python
class MessageComponent:
    def __init__(self, ...):
        self._render_cache = {}
        self._cache_timestamp = 0
        
    def __rich_console__(self, console, options):
        current_time = console.get_time()
        
        # Cache static states
        if self._state in (ComponentState.SUCCESS, ComponentState.ERROR):
            cache_key = (self._state, self._static_text)
            if cache_key in self._render_cache:
                yield self._render_cache[cache_key]
                return
                
        # Render and cache if needed
        rendered = self._render_current_state(current_time)
        if self._state != ComponentState.IN_PROGRESS:
            self._render_cache[cache_key] = rendered
        yield rendered
```

**Performance Strategies**:
1. **Cache static states**: Success/error states don't change, cache their renders
2. **Minimize allocations**: Reuse Text objects where possible
3. **Lazy calculations**: Only calculate shimmer when enabled
4. **Bounded history**: Keep last 5 words to avoid repeats, not entire history

## Key Technical Decisions

1. **No Internal Timers**: Use Rich's refresh cycle exclusively
2. **Text Spans for Shimmer**: Use Rich's Text class with character spans rather than raw ANSI
3. **Time-Based Animation**: Calculate positions from elapsed time for smooth, framerate-independent animation
4. **Property-Based API**: Use Python properties for clean state access (matching existing components)
5. **Caching Strategy**: Cache only static states, not animated ones

## Risk Mitigation

1. **Performance Risk**: Shimmer might be CPU intensive
   - **Mitigation**: Profile early, implement caching, consider reducing shimmer width if needed

2. **Timing Accuracy**: Word changes might not be precise
   - **Mitigation**: Use monotonic time, accept timing variance as feature not bug

3. **Terminal Compatibility**: Shimmer colors might not work everywhere
   - **Mitigation**: Rich handles fallbacks, test on multiple terminals
