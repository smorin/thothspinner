# Milestone M15: Progress Bar Format & Animation Smoothing (v1.1.0)

## Goal/Requirement
Extend the Textual ProgressWidget with a visual bar display format and smooth animated transitions.

These features were deferred from M09 (M09-T04 and M09-T09) to maintain strict Rich ProgressComponent feature parity. This milestone adds them as Textual-only enhancements.

**Sub-Requirements:**
- Add a "bar" format style to ProgressWidget that renders a visual progress bar using fill characters
- Add smooth animation when progress values change (eased transitions rather than instant jumps)
- Maintain full backward compatibility with existing ProgressWidget API

**Out of Scope:**
- Changes to the Rich ProgressComponent (Rich stays text-only)
- Changes to other Textual widgets (Spinner, Hint, Timer, Message)
- New widget classes (this extends the existing ProgressWidget)

### Feature 2: Animation Smoothing

#### Implementation Tasks
- [x] [M15-T06] Add constructor parameters for animation:
  - `animate: bool = False` — whether to animate value transitions
  - `animation_duration: float = 0.3` — transition duration in seconds
  - `animation_easing: str = "linear"` — easing function name (linear, ease_in, ease_out, ease_in_out)
- [x] [M15-T07] Add internal animation state:
  - `_display_current: float` — the visually displayed value (may differ from `current` during animation)
  - `_animation_timer: Timer | None` — Textual timer for frame updates
  - `_animation_start_value: float` — value at animation start
  - `_animation_target_value: float` — value animation is heading toward
  - `_animation_start_time: float` — monotonic time when animation began
- [x] [M15-T08] Implement easing functions as static methods:
  - `_ease_linear(t)` → `t`
  - `_ease_in(t)` → `t * t`
  - `_ease_out(t)` → `t * (2 - t)`
  - `_ease_in_out(t)` → smooth cubic
- [x] [M15-T09] Override `watch_current()` to start animation instead of immediate refresh:
  - If `animate=False` (default), call `refresh()` as before (backward compatible)
  - If `animate=True`, record start/target values, start timer at ~60fps
  - Timer callback: interpolate `_display_current`, call `refresh()`, stop timer when complete
- [x] [M15-T10] Update `_format_progress()` to use `_display_current` when animating:
  - For text formats (fraction, count_only, ratio, of_text): use `int(round(_display_current))`
  - For percentage format: use `_display_current / total * 100`
  - For bar format: use `_display_current / total` ratio for fill calculation
- [x] [M15-T11] Handle edge cases:
  - Rapid successive updates (interrupt current animation, start new one from current display value)
  - State transitions during animation (cancel animation, show state text immediately)
  - Widget unmount during animation (cancel timer in `on_unmount`)
  - `reset()` should cancel animation and jump to 0 immediately
- [x] [M15-T12] Update `from_config()` to accept animation keys

## Deliverable
```python
from thothspinner.textual.widgets import ProgressWidget
from textual.app import App, ComposeResult
import asyncio

class MyApp(App):
    def compose(self) -> ComposeResult:
        # Bar format with animation
        self.progress = ProgressWidget(
            current=0,
            total=100,
            format_style="bar",
            bar_width=30,
            bar_filled="█",
            bar_empty="░",
            bar_brackets=True,
            bar_show_percentage=True,
            animate=True,
            animation_duration=0.5,
            animation_easing="ease_out",
            color="#55FF55",
        )
        yield self.progress

    async def on_mount(self) -> None:
        for i in range(0, 101, 10):
            self.progress.set(i)       # Animates smoothly to each value
            await asyncio.sleep(1)
        self.progress.success()
        # Output at 40%: [████████████░░░░░░░░░░░░░░░░░░] 40%
```
